# 実装の優先順位

## 最優先方針

このサイトでは、以下を優先します。

1. 軽い
2. 見やすい
3. 壊れにくい
4. 編集しやすい
5. GitHub Pagesで公開しやすい

## 優先すること

- シンプルなHTML構造
- 分かりやすいCSS
- 最小限のJavaScript
- スマホで読みやすいレイアウト
- 後から内容を差し替えやすい構造
- 静的ファイルだけで完結する構成

## 優先しないこと

初期段階では、以下を優先しません。

- 高度なアニメーション
- 複雑なUI
- フレームワーク導入
- ビルド環境構築
- 管理画面
- CMS
- 動的ルーティング
- 外部API連携

## 判断基準

迷った場合は、次の基準で判断してください。

```text
より軽い方
より分かりやすい方
より壊れにくい方
よりGitHub Pagesで動きやすい方
```
