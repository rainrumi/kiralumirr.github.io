# HTML実装ルール

## 基本方針

HTMLは、意味が分かるセマンティックな構造にしてください。

## 使用する主なタグ

積極的に使用するタグ：

- `header`
- `main`
- `section`
- `article`
- `nav`
- `footer`
- `h1`
- `h2`
- `h3`
- `ul`
- `li`

## divの扱い

`div` は必要な場合のみ使ってください。  
ページ全体を `div` だけで構成しないでください。

## 見出し構造

`h1` は1ページに1つだけ使ってください。

推奨構造：

```text
h1: サイト名またはメインタイトル
h2: Profile / Works / Links などの主要セクション
h3: 作品カードや小項目のタイトル
```

## 仮テキスト

未確定の文章は `TODO` を付けてください。

例：

```html
<p>TODO: ここにプロフィール文を入れる。</p>
```

## リンク

リンクテキストは、リンク先の意味が分かる文言にしてください。

良い例：

```html
<a href="#">GitHubを見る</a>
```

悪い例：

```html
<a href="#">こちら</a>
```

外部リンクには必要に応じて以下を付けてください。

```html
target="_blank" rel="noopener noreferrer"
```
